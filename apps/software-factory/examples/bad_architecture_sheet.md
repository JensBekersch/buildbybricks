# Schlechtes Architecture Sheet Beispiel

Wir bauen einfach eine Django-App mit Kunden und Angeboten. Das wird mit Models,
Views und Templates gemacht. Spaeter kann man noch PDFs und Freigaben ergaenzen.

Warum dieses Beispiel schlecht ist:

- kein `schema_version`
- kein maschinenlesbares JSON
- keine expliziten Architecture Drivers
- keine Architekturentscheidungen mit Begruendung
- keine Akzeptanzkriterien
- keine Risiken mit Mitigation
- keine offenen Fragen
- kein Readiness-Status
- keine stabile Grundlage fuer Workorder-Agenten
